그리드 Grid

그리드는 대량의 데이터를 표현하기 위해 가장 많이 사용되는 컴포넌트입니다. SlickGrid 기반하에 AngularJS의 데이터 바인딩과 지시자

Usage

그리드 컴포넌트는 기본적으로 아래와 같이 태그를 입력하여 적용합니다.

parameter

그리드 컴포넌트의 주요 파라메터는 아래와 같습니다.

그리드 컨터에너 설정 정보(div)
egov-grid
그리드 컴포넌트를 명시합니다. 해당 속성이 적용된 요소는 꼭 Table을 포함하고 있어야 합니다. 이는 해당 요소가 그리드 컨터이너가 됨을 나타냅니다.
name
그리드 명을 나타냅니다. 해당 그리드 이름으로 컨트롤러에서 그리드 자바스크립트 인스턴스를 가지고 올 수 있습니다. 관련 내용은 그리드 서비스를 참고하세요.
dataset
그리드와 연결될 데이타 셋을 정의합니다. 데이타 셋의 포멧을 아래와 같아야 합니다.
[{
  columnNm1 : "value",
  columnNm2 : "value",
  ...
},{
  columnNm 1: "value",
  columnNm2 : "value",
  ...
},
 ...]
row-height
optional
그리드 로우의 높이를 나타냅니다. 숫자를 입력해야 합니다.
header-row-height
optional
그리드 해더 로우 높이를 명시합니다. 숫자를 입력해야 합니다.
enable-column-reorder
optional
컬럼을 정열 할 수 있는 여부를 명시합니다. boolean으로 입력해야 합니다.
enable-async-post-render
optional
비동기 컬럼 렌더링 여부를 명시합니다. boolean으로 입력해야 합니다.
그리드 셀 설정 정보(td)
width
optional
셀의 넓이를 명시합니다. 숫자를 입력해야 합니다.
resizable
optional
셀의 크기 변경 여부를 명시합니다. boolean으로 입력해야 합니다.
sortable
optional
셀의 정렬 여부를 명시합니다. boolean으로 입력해야 합니다.
formatter
optional
셀의 포멧팅을 담당하는 포멧터 함수의 이름을 명시합니다. 해당 포멧터는 gridProvider를 통하여 정의할 수 있습니다.(예제 참고)
class
optional
셀에 추가될 클래스를 정의합니다. 각 컬럼별로 해당 클래스가 들어가게 됩니다. css에 정의된 클래스 이름을 속성값으로 주어야 합니다.
async-post-render
optional
셀이 비동기로 그릴때 호출할 함수 이름을 명시합니다. 해당 함수는 $scope에 정의되어야 합니다. 또한 그리드 컨테이너에 enable-async-post-render가 true 되어있어야 작동합니다. (예제 참고)
